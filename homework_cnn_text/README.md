# Домашнее задание: CNN для классификации новостных текстов

## Задача

Многоклассовая классификация новостей Lenta.ru по рубрикам с помощью свёрточной нейросети (PyTorch).

## Источник данных

**Сайт:** [Lenta.ru](https://lenta.ru) — русскоязычное новостное издание.

**Способ сбора:** библиотека `requests` + парсинг RSS-лент и HTML-страниц статей (`BeautifulSoup`).

**URL-ы RSS по рубрикам:**
- Спорт: `https://lenta.ru/rss/news/sport`
- Экономика: `https://lenta.ru/rss/news/economics`
- Наука: `https://lenta.ru/rss/news/science`
- Культура: `https://lenta.ru/rss/news/culture`

## Разметка

**Автоматическая** — метка класса берётся из рубрики RSS-ленты, в которой опубликована статья (структура сайта):

| Класс | Метка | Описание |
|-------|-------|----------|
| 0 | sport | Спорт |
| 1 | economics | Экономика |
| 2 | science | Наука |
| 3 | culture | Культура |

Каждый пример: заголовок + первые 3 абзаца статьи.

## Представление текста: word-level CNN

Выбран **word-level** подход (токенизация по словам), а не character-level:

1. Новости содержат осмысленные слова и устойчивые n-граммы («центральный банк», «чемпионат мира»), которые CNN улавливает через свёртки размером 3–5.
2. Объём корпуса (~700 примеров) достаточен для словаря из частых слов без чрезмерного разреживания.
3. Character-level лучше для опечаток и морфологии, но требует больше данных; для тематической классификации по рубрикам word-level даёт более интерпретируемые признаки.

## Архитектура модели

```
Embedding(128) → Conv1D(k=3,4,5) × 3 → ReLU → GlobalMaxPool → Concat → Dropout(0.5) → Linear(4)
```

Несколько параллельных свёрток с разным размером ядра — классический TextCNN (Kim, 2014).

## Гиперпараметры и обоснование

| Параметр | Значение | Почему |
|----------|----------|--------|
| `MAX_SEQ_LEN` | 120 | Заголовок + 2–3 абзаца укладываются; длиннее — лишняя память |
| `MIN_WORD_FREQ` | 2 | Отсекаем опечатки и единичные токены |
| `EMBED_DIM` | 128 | Компромисс между ёмкостью и переобучением на ~700 примерах |
| `NUM_FILTERS` | 96 | Меньше 256 — меньше риск переобучения на малом корпусе |
| `KERNEL_SIZES` | (3, 4, 5) | Захват уни-, би- и триграмм на уровне слов |
| `DROPOUT` | 0.5 | Стандарт для текстовых CNN, снижает переобучение |
| `BATCH_SIZE` | 32 | Стабильные градиенты при ~500 train-примерах |
| `LEARNING_RATE` | 1e-3 | Типичное значение для Adam на CNN |
| `WEIGHT_DECAY` | 1e-4 | L2-регуляризация |
| `NUM_EPOCHS` | 15 | С early stopping (patience=4) |
| `PATIENCE` | 4 | Остановка при росте val loss |

- **Train:** 70%

Стratified split — пропорции классов сохраняются.

## Анализ результатов

**Что получилось хорошо:**
- Модель уверенно отличает «sport» (F1=0.89) — спортивная лексика хорошо выделяется свёртками.
- «science» имеет высокий recall (0.90) — научные термины часто встречаются только в этой рубрике.
- Обучение на CPU занимает ~20 секунд, пайплайн полностью воспроизводим.

**Что можно улучшить:**
- «economics» — самый слабый класс (F1=0.71): экономические новости иногда пересекаются по лексике с политикой и наукой.
- Класс «culture» меньше по объёму (140 vs 200) — можно собрать больше примеров для баланса.
